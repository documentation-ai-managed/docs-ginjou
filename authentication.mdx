## Overview

ginjou provides robust authentication and permission controls to secure your documentation space. You control access through login methods, user roles, team invitations, and API tokens. Start by setting up your account login, then manage team members and generate tokens for programmatic access.

<Callout kind="info">
  Enable two-factor authentication (2FA) on your account for enhanced security.
</Callout>

## Account Login Methods

Choose from multiple login options based on your needs.

<Tabs>
  <Tab title="Email/Password" icon="mail">
    Create an account with your email and a strong password.

    <Steps>
      <Step title="Sign Up" icon="user-plus">
        Visit `https://dashboard.example.com/signup` and enter your details.
      </Step>
      <Step title="Verify Email">
        Check your inbox for the verification link.
      </Step>
      <Step title="Login">
        Use your credentials at `https://dashboard.example.com/login`.
      </Step>
    </Steps>
  </Tab>
  <Tab title="OAuth (Google)" icon="chrome">
    Connect with Google for seamless single sign-on.

    ```javascript
    // Redirect to OAuth endpoint
    window.location.href = 'https://auth.example.com/oauth/google?redirect_uri=https://dashboard.example.com/callback';
    ```
  </Tab>
  <Tab title="OAuth (GitHub)" icon="github">
    Authorize via GitHub.

    ```bash
    curl -X POST https://auth.example.com/oauth/github \
      -d "client_id=YOUR_CLIENT_ID" \
      -d "redirect_uri=https://dashboard.example.com/callback"
    ```
  </Tab>
</Tabs>

## User Roles and Permissions

Assign roles to control access levels. Each role has predefined permissions.

<Columns cols={2}>
  <Card title="Admin" icon="shield" horizontal>
    Full access: edit docs, manage users, settings.
  </Card>
  <Card title="Editor" icon="edit-3" horizontal>
    Edit and publish docs, no user management.
  </Card>
  <Card title="Viewer" icon="eye" horizontal>
    Read-only access to docs.
  </Card>
  <Card title="Guest" icon="user-x" horizontal>
    Temporary view access, expires after 30 days.
  </Card>
</Columns>

| Role    | Edit Docs | Manage Users | Delete Content | View Analytics |
|---------|-----------|--------------|----------------|---------------|
| Admin   | ✅        | ✅           | ✅             | ✅            |
| Editor  | ✅        | ❌           | ❌             | ✅            |
| Viewer  | ❌        | ❌           | ❌             | ❌            |
| Guest   | ❌        | ❌           | ❌             | ❌            |

## Team Invitations and Management

Invite collaborators to your ginjou space.

<Steps>
  <Step title="Invite User" icon="user-plus">
    Go to Team Settings > Members > Invite.
  </Step>
  <Step title="Set Role">
    Select role and permissions during invitation.
  </Step>
  <Step title="Manage Access">
    ```javascript
    // API example to update user role
    await fetch('https://api.example.com/v1/teams/{team_id}/members/{user_id}', {
      method: 'PATCH',
      headers: {
        'Authorization': `Bearer ${YOUR_TOKEN}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({ role: 'editor' })
    });
    ```
  </Step>
</Steps>

## API Access Tokens

Generate tokens for API integration. Tokens inherit your user's permissions.

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token format: `Bearer YOUR_TOKEN`.
</ParamField>

<ParamField query="team_id" param-type="string" required="false">
  Optional team identifier for scoped access.
</ParamField>

<CodeGroup tabs="JavaScript,Python">
  ```javascript
  const response = await fetch('https://api.example.com/v1/docs', {
    headers: {
      'Authorization': `Bearer YOUR_TOKEN`
    }
  });
  const docs = await response.json();
  ```
  ```python
  import requests
  headers = {'Authorization': 'Bearer YOUR_TOKEN'}
  response = requests.get('https://api.example.com/v1/docs', headers=headers)
  docs = response.json()
  ```
</CodeGroup>

<Expandable title="Regenerate or Revoke Tokens" default-open="false">
  In Dashboard > API Tokens, click Regenerate to get a new token or Revoke to invalidate.
</Expandable>

<Callout kind="alert">
  Never expose tokens in client-side code or public repos. Rotate tokens regularly.
</Callout>
